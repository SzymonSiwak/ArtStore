# ARTSTORE - SOFTWARE ARCHITECTURE DOCUMENT (SAD)

**Project:** ARTSTORE - Homepage & Category Structure Refactoring  
**Status:** Artstore (v1.1)  
**Tech Stack:** .NET / C#, Entity Framework Core, SQL, Web API, Blazor

## 1. The aim of the project
This document outlines the architectural changes required to support the new homepage layout and the redesigned product categorization logic for the Artstore web application. The goal is to transition from a static, hardcoded categorization system to a dynamic, database-driven domain model.

**Scope of changes:**
* Implementation of new homepage graphic design.
* Retention of existing sections: Bestsellers, New Arrivals.
* Removal of sections: Living, Stationary.
* Introduction of a new product section: Artworks.
* Restructuring of product taxonomy by introducing two independent navigational axes: Categories and Collections.

## 2. Architectural drivers
* **Dynamic Data Management:** Content administrators require the ability to manage categories and collections dynamically via the CMS/Database, rather than relying on application redeployments (replacing code-level enums).
* **Complex taxonomy:** Products now require multi-dimensional classification (Categories and Collections) with cross-filtering capabilities.
* **SEO Optimization:** Routing must transition to user-friendly and SEO-compliant URLs utilizing slugs.

## 3. Architectural Decisions

### DOMAIN MODEL & DATABASE ARCHITECTURE

* **Deprecated Structure**
  * `Category` Enum: To be entirely removed from the domain layer. All references in existing entities and logic must be refactored.
* **New entities**
  Two new SQL database tables/entities will be introduced to the Entity Framework Core context:
  * `Category` entity
    Properties: `Id` (PK), `Name`, `Description`, `Slug` (Unique Index).
  * `Collection` entity
    Properties: `Id` (PK), `Name`, `Description`, `BannerImageUrl`, `Slug` (Unique Index).
* **Entity Relationship**
  The `Product` entity must be updated to establish relationships with the newly created tables:
  * Product to Category (1:M / One-to-Many): A Product must belong to exactly one Category. The Category table will have a One-to-Many relationship with the Product table.
  * Product to Collection (M:M / Many-to-Many): A Product can fit various styles and belong to multiple Collections. A linking table (e.g., `ProductCollection`) must be generated by EF Core to support this.

### APPLICATION LAYER (BACKEND / API)

* **Data transfer Object (DTOs)**
  New DTOs must be created to decouple the database entities from the API response:
  * `CategoryDto`: `Id`, `Name`, `Slug`
  * `CollectionDto`: `Id`, `Name`, `Slug`, `BannerImageUrl`
* **Query and Command Updates (CQRS/Services)**
  HomePage Data: Update `GetHomePageDataQuery`
  * Remove: Logic fetching data for "Living" and "Stationary".
  * Add: Logic to fetch `FeaturedCollections` and the new "Artworks" arrivals.
  * Update `HomePageDto` accordingly to reflect these payload changes.
* **Repository Logic Refactoring**
  The `ProductRepository` will undergo significant consolidation to support the new cross-filtering requirements.
  * Deprecated Methods: Remove isolated fetching methods such as `GetBestsellersAsync()` and `GetByCategoryAsync()`.
  * Unified Search Method: All listing and filtering requests will be routed through an updated `GetFilteredProductsAsync()` method.
  * `ProductFilterDto` Updates:
    * Remove: `CollectionName`
    * Add: `CategoryId`, `CollectionId`, `IsBestseller`, `IsNewArrival`, `CategorySlug`, `CollectionSlug`.

### PRESENTATION LAYER (FRONTEND / BLAZOR)

* **Routing & URL Strategy**
  The Blazor application will implement a route-based architecture driven by dynamic parameters (slugs) rather than internal GUIDs/IDs for main listing pages.
  * Route Definition: Implement `@page "/category/{CategorySlug}"` and `@page "/collection/{CollectionSlug}"`.
  * Component Behavior: The Blazor component will parse the URI parameter (`CategorySlug`), call the Backend API to resolve the corresponding database `CategoryId`, and then issue the primary query to `GetFilteredProductsAsync` using the resolved ID as the base filter.
* **UI Components & State**
  * New Layout: Implement the new homepage and navbar layouts based on the latest Figma specifications.
  * Cross-Filtering UI: Within a Collection subpage, implement a UI filter component allowing users to refine the current view by selecting a specific Category (and vice versa).
  * Cleanup: Remove all Blazor pages, components, and routing related to the deprecated "Living" and "Stationary" sections. Add frontend services for fetching category and collection lists from the API.
